JavaScript OOP Component

Reusable Counter with State-Driven UI

This project demonstrates a reusable counter component built with JavaScript classes, dynamic DOM creation, and UI updates that always reflect the current state.

Project Highlights

  • Reusable Counter class
  • State-driven interface updates
  • Multiple counters on one page
  • StepCounter subclass with custom step values

Overview

What the Counter class does

The Counter class creates a reusable UI component that builds its own display and control buttons directly inside a selected container. Each counter instance stores its own count value and updates its interface whenever that value changes.

The component solves the problem of repeating the same counter markup and logic multiple times. Instead of writing each counter manually in HTML, a new counter can be created with a selector and an optional starting value.

It also demonstrates state-driven UI behavior by visually changing the decrement and reset buttons when the count reaches zero.

What problem it solves

Without a reusable class, every counter would need its own hard-coded markup, event listeners, and update logic. This project packages all of that into a single component so the same logic can be reused multiple times on the page with different starting values.

The StepCounter subclass extends that base behavior and shows how the same component can be customized through inheritance and polymorphism.

How To Use It

Set up the component step by step

Step 1: Add empty containers in your HTML

The counter UI is not written directly in HTML. Your JavaScript class creates the display and buttons inside a container, so each counter instance only needs an empty container div. 

<div id="counter-container1" class="counter demo-card"></div>
        <div id="counter-container2" class="counter demo-card"></div>
        <div id="counter-container3" class="counter demo-card"></div>
        <div id="counter-container4" class="counter demo-card"></div>

Each id must match the selector passed into the JavaScript constructor.

Step 2: Import the classes in main.js

Your project exports both the base Counter class and the StepCounter subclass from the same module file. 

import { Counter } from "./modules/counter.js";
        import { StepCounter } from "./modules/counter.js";

Step 3: Create standard Counter instances

The base Counter constructor accepts two values:

  • selector — the container where the component mounts
  • initialValue — the starting count value
const counter1 = new Counter("#counter-container1", 0);
        const counter2 = new Counter("#counter-container2", 6);
        const counter3 = new Counter("#counter-container3", 8);

In your current setup:

  • counter1 starts at 0
  • counter2 starts at 6
  • counter3 starts at 8

Step 4: Create a StepCounter instance

The StepCounter extends Counter and accepts a third parameter: step. This changes how much the count increases or decreases each time a button is clicked. 

const counter4 = new StepCounter("#counter-container4", 5, 10);

This means:

  • "#counter-container4" is the container selector
  • 5 is the starting count
  • 10 is the step value

So the count changes like this: 

Start: 5
        Increment: 15 → 25 → 35
        Decrement: 25 → 15 → 5 → 0

If the step value is larger than the remaining count, the class clamps the result back to 0 so the counter never goes negative.

Step 5: Understand how the UI is created

Inside the mount() method, the class selects the container and dynamically creates the counter interface. 

this.display = document.createElement("div");
        this.incrementButton = document.createElement("button");
        this.decrementButton = document.createElement("button");
        this.resetButton = document.createElement("button");

The buttons are then appended into the selected container and connected to the class methods through event listeners.

Step 6: Understand how the UI updates

Every time the count changes, the class calls update(). This method handles both the visible number and the inactive state of the buttons. 

update() {
        this.display.textContent = `Count: ${this.count}`;

        this.decrementButton.classList.toggle("inactive", this.count === 0);
        this.resetButton.classList.toggle("inactive", this.count === 0);
        }

When the count reaches 0, the decrement and reset buttons receive the inactive class so the UI clearly reflects the current state.
